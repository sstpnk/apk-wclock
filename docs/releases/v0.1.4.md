# WClock v0.1.4

Patch release focused on legacy Android render pacing.

## Changes

- Continuous rendering now uses Android vsync invalidation on API 16+ instead of a fixed 33 ms handler delay.
- Weather status redraws use the same vsync invalidation path.
- Photo frame mode now prepares bitmaps in the background instead of decoding inside the draw frame.
- Temporary flicker diagnostic switches are removed from the release settings screen.
- Retired photo bitmaps are kept alive briefly before recycling to avoid pending hardware renderer references.
