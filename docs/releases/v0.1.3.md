# WClock v0.1.3

Patch release focused on long-running photo wall smoothness.

## Changes

- Photo wall decoding is moved out of the draw frame and into a background prefetch step.
- The next photo wall bitmap is prepared before the configured display interval elapses, so entrance animation can start without waiting for storage I/O.
- Queued and stale prepared bitmaps are recycled safely when the photo source changes.
- Render settings are cached between frame updates instead of being read from preferences every frame.
- Clock panel width is measured from actual text bounds and stays tighter on dense screens when seconds are enabled.
