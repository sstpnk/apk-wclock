# WClock v0.1.6

## Highlights

- Security hardening: wttr.in now uses HTTPS and the app no longer allows cleartext traffic globally.
- Performance cleanup: the clock render path no longer allocates a new `Calendar` object every frame.
- Runtime cleanup: weather refresh uses a single executor instead of creating a new thread per refresh.
- APK-size-oriented cleanup: removed dead code, unused fields, duplicated weather repository construction, and unused resources.
- Icon decision: reverted the vector icon experiment and kept the original launcher PNG assets because the release APK size difference was negligible.
- Repository cleanup: README is now a bilingual product page and the license is now standard MIT.
